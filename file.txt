
**Success Indicator:** You will see a local URL, usually `http://localhost:5173`.

1. Open your web browser (Chrome, Firefox, Edge, etc.).

## Troubleshooting

### Issue: "Failed to fetch" Error
**Cause:** The browser is blocking the connection to Ollama (CORS).
**Solution:**
- Ensure you followed **Step 3** above to set `OLLAMA_ORIGINS="*"`.
- If you just set the variable, make sure you have **restarted** the `ollama serve` process.
- Try refreshing the web page using `Ctrl + Shift + R` (Hard Refresh) to clear the cache.

### Issue: "Connection Refused" or "ECONNREFUSED"
**Cause:** The Ollama server is not running.
**Solution:**
- Check your first terminal window. Is `ollama serve` active?
- If it crashed, restart it.
- Ensure no other application is using port `11434`.

### Issue: "Model not found"
**Cause:** You haven't downloaded the model yet.
**Solution:**
- Run `ollama pull llama3.1` in your terminal.
- Wait for the download to complete (it may take a few minutes depending on your internet speed).

### Issue: The UI looks broken / No styles
**Cause:** Tailwind CSS might not be configured correctly.
**Solution:**
- Ensure you ran `npm install`.
- Check your terminal for errors during `npm run dev`.
- Try stopping the server (`Ctrl + C`) and running `npm run dev` again.

### Issue: Response is very slow
**Cause:** Running AI models locally depends on your computer's hardware (CPU/GPU/RAM).
**Solution:**
- Ensure you aren't running other heavy applications.
- If you are on a laptop, check if you are running on battery (plugging in often improves performance).
- Try a smaller model (e.g., `phi3` or `gemma2:2b`) if your hardware is limited.

## Need Help?

If you are still stuck, please check:
1. That Node.js and Ollama are correctly installed.
2. That both terminals (Ollama and React) are running without errors.
3. That you have set the `OLLAMA_ORIGINS` environment variable.
